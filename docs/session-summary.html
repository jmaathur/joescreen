M9 Receive-side lifecycle correctness + share metadata

Made movable window shares production-grade and landed the shared contracts M10/M11 build on.

  • SID-keyed transport registry (fixes latent bug #1: two "camera" tracks no longer overwrite each other).

  • Owner repair (latent #5); aspect-true windows + cursor letterbox mapping via VideoFitMath; WindowResizeWatcher; ShareInfo.

M10 Participant video tile strip

The "see everyone" ask.

  • Pure TrackClassifier routing; display names via the JWT name claim (SFU-distributed to late joiners, zero wire surface).
  • ParticipantMediaReducer — mic/camera state reads publication.isMuted, never track subscription (a muted camera would otherwise render a frozen frame).
  • The tile strip + live share thumbnails (a second renderer on the held track); decode budget with shares taking priority.
  • Adversarial review caught 2 real R24/R32 bandwidth bugs — the thumbnail defeating soft-hide, and the decode budget over-counting closed windows. Both fixed.

M11 Share the whole screen (display share)

  • D5-structural codec-ordering fix (latent #3) via the pure ShareContext; admission revival (dead code #4) with ShareBitratePolicy.
  • Display picker + macOS-14-floor DisplayPickResolver; DisplayCaptureService (hall-of-mirrors filter, screen-lock pause, display-unplug).

Adversarial review paid off

Before each milestone commit, 3+ reviewers with distinct lenses (Swift-6 concurrency, wire back-compat, R24/R32 subscription correctness) audited the diff, then each finding was adversarially verified. This surfaced 6 confirmed defects — mostly concurrency correctness bugs that unit tests alone would never have caught:
  • M10: share thumbnail's second renderer defeated the soft-hide (kept the SFU forwarding).
  • M10: decode budget counted user-closed (zero-decode) windows, starving camera tiles.
  • M11: a display-lifecycle watcher claimed @unchecked Sendable with no lock — a real data race on teardown.
  • M11: codec renegotiation republished a stale entry if the window was concurrently unshared — an orphaned live "ghost" track.
  • M11: admission .degrade never lowered already-live tracks' bitrates.
  • M11: a share refused by the encode cap flickered a live window through H.264 and back.

Hard constraints honored

  • R22 one-libwebrtc — only JoeScreenLiveKit links LiveKit; JoeScreenKit stayed LiveKit- and CoreMedia-free.
  • R24 / R32 — adaptiveStream+dynacast kept on; never set(enabled:); renderers report adaptive-stream size.
  • D5 — VP9 for a single window, H.264 for ≥2 windows or any display, enforced structurally via ShareContext.
  • D12 — input/terminal authorization is owner-side against trusted local state; coordination-plane mirrors are display-only.
  • Wire protocol — every change additive: new kinds appended (never renumbered), new fields optional + decodeIfPresent, no receiver-local revision bumps.
